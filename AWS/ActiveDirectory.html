Creating a Virtual Private Cloud (VPC)

April 14, 2021 by Maximilian Lever

  1. Open VPC
  2. Launch VPC Wizard
  3. Select VPC with a Single Public Subnet
  4. In VPC name enter
    EXAMPLE-VPC
  5. Select the Availability Zone
    eu-west-2a
  6. Create VPC
  7. Select Subnet
  8. Create Subnet
  9. Select
    EXAMPLE-VPC
  10. Under Subnet Name enter
    EXAMPLE-VPC1
  11. Select same Availability Zone as in step 5
  12. In IPv4 CIDR block enter
    10.0.0.0/24
  13. Create subnet
  14. Repeat the steps to create a subnet
  15. Under Subnet Name enter
    EXAMPLE-VPC2
  16. Make sure you have selected a different Availability Zone to the previous subnet
  17. In IPv4 CIDR block enter
    10.0.1.0/24
  18. Create subnet

Creating a Directory

April 14, 2021 by Maximilian Lever

  1. Open Directory Service
  2. Set up directory
  3. Select AWS Managed Microsoft AD
  4. Select Standard Edition
  5. In Directory DNS name enter
    www.example.co.uk
  6. In NetBIOS enter
    ex
  7. In Directory description enter
    Example Active Directory
  8. Chose an Admin password and then Confirm password
  9. Click Next
  10. Select
    EXAMPLE-VPC
  11. Select
    EXAMPLE-VPC1
  12. Select
    EXAMPLE-VPC2
  13. Click Next
  14. Create Directory

Creating a Domain Controller Instance

April 14, 2021 by Maximilian Lever

  1. Open EC2
  2. Launch Instance
  3. Select AMI Microsoft Windows Server 20xx Base
  4. Select t2.micro
  5. Next: Configure Instance Details
  6. Under Network Select
    EXAMPLE_VPC
  7. Under Subnet Select
    EXAMPLE_VPC1
  8. Enable Auto Assign Public IP Address
  9. In Domain join directory Select
    Example Directory
  10. In IAM role select
    EC2DomainJoin
  11. Click Next to Configure Security Group
  12. Make sure there is an RDP rule with Source set to Anywhere
  13. Review and Launch

Connecting to Domain Controller Instance

April 14, 2021 by Maximilian Lever

  1. Open Instances
  2. Select the Instance you just Launched
  3. Click Connect
  4. RDP Client
  5. Download remote desktop file
  6. When given the option to connect to the server click Connect
  7. Click more choices
  8. Use different account
  9. In username enter
    ex\admin
  10. In password enter the password you entered when setting up the directory
  11. Login
