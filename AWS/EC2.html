Creating an AMI

September 12, 2020 by Maximilian Lever

  1. Open EC2 from AWS Management console.
  2. Go to running instances and select the instance you want to make an image of.
  3. Select Actions>Image>Create Image.
  4. Name the image.
  5. Give description that best fits the image.
  6. Deselect No reboot.
  7. Finish by Clicking Create image.

Connecting via PuTTY

September 9, 2020 by Maximilian Lever

  1. Download and install PuTTY from https://www.chiark.greenend.org.uk/~sgtatham/putty/latest.html.
  2. Open PuTTYgen.
  3. Click Load and select the .pem file.
  4. Click Save private key.
  5. Click Yes to saving the key without a passphrase.
  6. Name the .ppk file the same as the .pem file.
  7. Open PuTTY and enter the Public IP address of the server.
  8. In Category under Connection > SSH > Auth browse for the .ppk file that was generated.
  9. Click Open.
  10. When asked ‘login as:’ type in ec2-user.
    login as: ec2-user

Connecting via Gitbash

September 9, 2020 by Maximilian Lever

  1. Open a Gitbash terminal where the keypair to the server is.
  2. Type into the terminal
    ssh -i keypair.pem ec2-user@public-dns.com.

Sending files via SCP

September 9, 2020 by Maximilian Lever

  1. Open a Gitbash terminal where the keypair to the server is.
  2. Type into the terminal
    scp -i keypair.pem file.html ec2-user@public-dns.com:~/.
  3. Type into the terminal
    ssh -i keypair.pem ec2-user@public-dns.com.
  4. Move file to the the apache document root folder using.
    mv file.html /var/www/html.
