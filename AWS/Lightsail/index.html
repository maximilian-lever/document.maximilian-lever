AWS Lightsail

Create a WordPress instance

September 01, 2021 by Maximilian Lever

  1. Sign into the Lightsail console.
  2. On the Instances tab of the Lightsail home page, choose Create instance.
  3. Choose your instance image.
  4. Choose Linux/Unix as the platform.
  5. Choose WordPress as the blueprint.
  6. Choose 512mb, 1vCPU, 20GB SSD instance plan.
  7. Enter a name for your instance.
  8. Choose Create instance.

Get the password for your WordPress website

September 01, 2021 by Maximilian Lever

  1. On the Instances tab of the Lightsail home page choose the SSH quick-connect icon for your WordPress instance.
  2. After the browser-based SSH client window opens, enter the following command to retrieve the default application password:
    cat $HOME/bitnami_application_password
  3. Make note of the password displayed on the screen. You use it later to sign in to the administration dashboard of your WordPress website.

Sign in to dashboard of your WordPress website

September 01, 2021 by Maximilian Lever

  1. In a browser, go to:
    http://PublicIpAddress/wp-login.php
  2. Log into your instance.
  3. In the Username or Email Address box, enter "user".
  4. Password box, enter the default password obtained earlier in this tutorial.
  5. Choose Log in.

Create a Lightsail static IP address

September 01, 2021 by Maximilian Lever

  1. On the Instances tab of the Lightsail home page, choose your running WordPress instance.
  2. Choose the Networking tab, then choose Create static IP.
  3. The static IP location, and attached instance are pre-selected based on the instance that you chose earlier in this tutorial.
  4. Name your static IP, then choose Create.

Create a Lightsail DNS zone and map a domain

September 01, 2021 by Maximilian Lever

  1. On the Networking tab of the Lightsail home page, choose Create DNS zone.
  2. Enter your domain, then choose Create DNS zone.
  3. Make note of the name server address listed on the page.
  4. After management of your domain’s DNS records are transferred to Lightsail, add an A record to point the apex of your domain to your WordPress instance, as follows:
  5. In the DNS zone for your domain, choose Add record.
  6. In the Subdomain box, enter an @ symbol to map the apex of your domain (such as example.com) to your instance. The @ symbol explicitly symbolizes that you’re adding an apex record. It is not added as a subdomain.
  7. In the Maps to box, choose the static IP that you attached to the WordPress instance in the previous step of this tutorial.
  8. Choose the save icon.
    Allow time for the change to propagate through the internet's DNS before your domain begins routing traffic to your WordPress instance.
