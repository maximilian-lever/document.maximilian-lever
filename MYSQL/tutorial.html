MYSQL Tutorial

Janurary 22, 2021 by Maximilian Lever

In this tutorial I will be explaining how to create a database in MYSQL and standard practices.

Step 1:

In this step we are going to create a database called user. We will use the syntax below to create said database.

CREATE DATABASE user;

Step 2:

Within this database we will need a user table. The naming standard we will use for this table will include the object at the beginning of the name. E.g. tbl_user.

CREATE TABLE `tbl_user` (
`user_id` INT(11) NOT NULL AUTO_INCREMENT,
`username` VARCHAR(8) NOT NULL,
`password` VARCHAR(8) NOT NULL,
`first_name` VARCHAR(16) NOT NULL DEFAULT 'John',
`last_name` VARCHAR(16) NOT NULL DEFAULT 'Doe',
`update_time` DATETIME NOT NULL DEFAULT '2000/01/01 00:00:00',
`create_time` DATETIME NOT NULL DEFAULT '2000/01/01 00:00:00',
PRIMARY KEY (`user_id`)
);

In the syntax above after telling the database to CREATE the user table we will list what columns we want in said table.

user_id :

Every table must have an ID column and the naming convention we will use is the name of the table followed by ‘_id’.

We will declare the datatype of the column being an INTERGER and the length of the data allowed in this column. E.g. INT(11) equals a number with 11 digits is the maximum allowed. (10,000,000,000)

We will set the rule that the column is not allowed to be NULL and by default it will AUTO_INCREMENT E.g 1, 2, 3, 4

username :

This column will store a user’s username, there is no naming convention for this column. Again we will declare the datatype as a VARIABLE CHARACTER and the maximum number of characters the column can hold. E.g VARCHAR(8)will hold a word with 8 characters.

password:

This column will be similar to the user column and have no naming convention, the data type is a VARIABLE CHARACTER and the number of characters is 8.

first_name:

This column will hold the user's first name. This is the first column where we will set if data isn’t inserted then it will by DEFAULT put the data in as John.

last_name:

This will hold the user's last name. and the DEFAULT is Doe.

update_time :

Here we will hold the time and date if this row is ever updated. E.g The user changes their password.

The DEFAULT here is 2000/01/01 00:00:00.

create_time :

Like the update column we have a create column for when the row is created.

The DEFAULT here is 2000/01/01 00:00:00.

Every table will require a primary column; a common practice is to set the ID column as the PRIMARY.

PRIMARY KEY (`user_id`)

Here we set user_id as the PRIMARY column.

Step 3:

To insert data into this row we will use the following syntax.

INSERT INTO `user`.`tbl_user` ( `username`, `password` ) VALUES ('doej', 'password');

The syntax listed above tells the database to ’INSERT INTO’ and then we list the database followed by the table within.

Then we list the columns that we want to put data into within brackets, folowed by VALUES and then the data within brackets.

Step 4:

The final part of this tutorial is retrieving the data.

SELECT * FROM tbl_user WHERE user_id = 1;

The syntax listed above tells the database to SELECT everything FROM the user table where the user_id for the row equals 1.

This will return:

user_id:1
username:doej
password:password
first_name:John
last_name:Doe
update_time:2000/01/01 00:00:00
create_time:2000/01/01 00:00:00
